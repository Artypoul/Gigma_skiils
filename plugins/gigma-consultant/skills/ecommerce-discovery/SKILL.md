# Выявление потребностей интернет-магазина (бриф)

Цель — понять бизнес клиента **прежде** чем предлагать Gigma, и зафиксировать факты в бриф, на который обопрутся `ecommerce-solution-fit` и `ecommerce-commercial-proposal`.

Каждый вопрос задаётся с целью — он питает конкретную возможность Gigma (карта — в [`reference/ecommerce-capabilities.md`](../../reference/ecommerce-capabilities.md)). Не зачитывай клиенту «зачем», просто спрашивай по-человечески.

## Как спрашивать

- **Одним блоком 6–8 вопросов**, не по одному (это уважение к времени клиента). Если интерфейс позволяет — кнопками-вариантами.
- Простым языком, без жаргона.
- Не уверен в ответе клиента — переспроси, не достраивай сам.
- Ответы сразу пиши в бриф (`Write` в файл `discovery-brief-<клиент>.md`).

## Что выяснить (и какую возможность Gigma это питает)

| Вопрос клиенту | Зачем (что в Gigma настраивается) |
|---|---|
| Что продаёте — товары, услуги или аренда? | вид номенклатуры: товар / услуга |
| Сколько примерно позиций (SKU)? Есть категории, бренды? | объём каталога, импорт прайса, структура категорий |
| Прайс уже есть файлом (Excel/1С)? В каком виде? | массовая загрузка `nomenclatures/import` (xlsx) или выгрузка из 1С |
| Сейчас где-то продаёте онлайн (Tilda, Insales, сайт, маркетплейсы)? | миграция данных, что переносим |
| Нужен учёт остатков и склад(ы)? Один склад или несколько? | warehouses + inventories; мультисклад |
| Как доставляете — самовывоз, курьер, по адресу? | delivery_types: самовывоз (нужен магазин-ПВЗ), доставка по адресу |
| Как хотите принимать оплату — картой онлайн, наличными, при получении? Есть кабинет YooKassa? | YooKassa (оплата картой) vs наличные; creds клиента |
| Сколько заказов в день ожидаете? Кто будет обрабатывать (сколько сотрудников, роли)? | нагрузка, роли/доступы сотрудников |
| Нужен личный кабинет клиента, избранное, история заказов? | контур counterparty |
| Нужны промокоды/скидки? | discounts |
| Контент: страницы (о доставке, оплате), баннеры, меню — кто наполняет? | CMS витрины (pages, slides, menus, blocks) |
| Что болит прямо сейчас? Какой результат через 1–2 месяца считаете успехом? | приоритеты для solution-fit и КП |
| Желаемые сроки и ориентир по бюджету? | планирование внедрения и КП |

## Шаблон брифа (сохранить файлом)

```markdown
# Бриф: <клиент> (интернет-магазин)
Дата: <дата>

## Бизнес
- Что продаёт: товары / услуги / аренда
- Ассортимент: ~N SKU, категорий ___, бренды: да/нет
- Прайс: формат ___ (Excel / 1С / нет файла)
- Сейчас онлайн: ___ (площадка / нет)

## Процессы
- Склад/остатки: нужен да/нет, складов ___
- Доставка: самовывоз / курьер / по адресу
- Оплата: карта онлайн (YooKassa: есть кабинет да/нет) / наличные / при получении
- Заказов в день: ~___
- Команда: сотрудников ___, роли ___

## Витрина и клиенты
- Личный кабинет / избранное / история: да/нет
- Промокоды/скидки: да/нет
- Контент (страницы, баннеры, меню): кто наполняет ___

## Цели и рамки
- Главная боль: ___
- Успех через 1–2 мес: ___
- Сроки: ___    Бюджет (ориентир): ___

## Флаги для solution-fit (заполняет консультант)
- [ ] что-то из запрошенного помечено ⚠/баг в карте возможностей → проговорить
```

## Что дальше

Бриф готов → переходи к скилу **ecommerce-solution-fit**: разложить эти ответы на конкретные возможности Gigma и честно отметить ограничения.
