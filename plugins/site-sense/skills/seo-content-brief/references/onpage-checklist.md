# On-page checklist

## Metadata

- [ ] Title is specific, readable, and not keyword-stuffed.
- [ ] Meta description explains value and next step.
- [ ] One H1 per page, aligned with primary intent.
- [ ] URL is stable, readable, lowercase, and hyphenated.

## Structure

- [ ] First screen or intro confirms the user is in the right place.
- [ ] H2/H3 sections follow user questions and decision order.
- [ ] The page does not mix incompatible intents.
- [ ] FAQ answers real doubts, not promotional filler.

## Content quality

- [ ] Claims are backed by examples, proof, data, screenshots, or process.
- [ ] Content includes concrete cases or scenarios.
- [ ] Definitions are included when needed, but not overexplained for expert intent.
- [ ] Product tie-in helps the user rather than interrupting.
- [ ] No generic intro like «в современном мире».

## Links

- [ ] Page links to relevant product/service pages.
- [ ] Related articles and supporting pages link back to hub page.
- [ ] Anchors describe the target page.
- [ ] Important pages are not orphaned.

## Schema candidates

- [ ] BreadcrumbList for nested pages.
- [ ] FAQPage when FAQ is substantial and accurate.
- [ ] Article/BlogPosting for editorial pages.
- [ ] Product/Service/LocalBusiness when appropriate and truthful.

## Conversion

- [ ] CTA matches the user's stage.
- [ ] Secondary CTA exists for low-intent visitors.
- [ ] Proof appears before high-friction CTA.
- [ ] Next step is clear after reading.
