# Proof gap audit

Use this to find places where copy makes promises without enough evidence.

## Audit table

```markdown
| Page section | Claim | Proof present? | Proof strength 1-5 | Gap | Fix |
|---|---|---|---:|---|---|
| Hero |  | yes/no |  |  |  |
```

## Proof placement rule

The stronger the claim, the closer the proof should be.

- Hero-level claim → proof strip or immediate example.
- Benefit claim → supporting example, number, or screenshot.
- Process claim → steps, timeline, owner, deliverables.
- Risk claim → guarantee, SLA, security note, limitation, or compliance proof.

## Common proof fixes

- Add a concrete case instead of a generic testimonial.
- Add before/after example.
- Replace «быстро» with scope and timing.
- Replace «экономит» with calculation method.
- Add screenshot or sample deliverable.
- Add process details if numbers are unavailable.
- Soften claim if proof cannot be added.

## Language for unsupported claims

Instead of absolute promises:

- «поможет сократить» instead of «сократит»;
- «обычно занимает» instead of «занимает всегда»;
- «подходит для» instead of «идеально для»;
- «проверяем на пилоте» instead of «гарантируем результат».
