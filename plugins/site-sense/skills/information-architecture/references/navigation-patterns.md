# Navigation patterns

Navigation must help users choose paths, not expose every internal page.

## Header

Best for high-priority paths:

- product/service overview;
- solutions/use cases;
- pricing;
- resources or guides;
- company/trust, if important;
- primary CTA.

### Rules

- Keep 3–7 primary items.
- Use labels users recognize.
- Put the main CTA visually and verbally apart from navigation.
- Do not hide core conversion pages only in footer.

## Dropdown / mega menu

Use when users need to choose by:

- use case;
- industry;
- product module;
- role;
- resource type.

Avoid dropdowns that mix unrelated groups.

## Footer

Footer should support exploration and trust:

- all important product/service pages;
- resources and guides;
- company pages;
- legal pages;
- contact/support;
- secondary languages/regions.

## Breadcrumbs

Use for:

- blogs/guides;
- catalogs;
- docs;
- nested service pages;
- case studies grouped by type.

Pattern:

```text
Home → Section → Page
```

Breadcrumbs should match information hierarchy, not necessarily every URL segment if URL is intentionally flatter.

## Contextual links

Add links inside body copy from:

- problem pages to solution pages;
- blog posts to product/service pages;
- comparison pages to pricing/demo;
- docs to related tutorials;
- case studies to relevant use cases.

## Label checks

- Would a new user understand this label without knowing the company?
- Are two labels too similar?
- Is this a destination label or an action label?
- Does the label match the page title and URL?
