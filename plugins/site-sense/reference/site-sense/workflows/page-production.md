# Workflow: производство отдельной страницы

Используй для главной, лендинга, страницы услуги, страницы продукта, страницы сравнения, pricing, lead magnet или SEO-страницы.

## Главное исправление цепочки

Нельзя начинать с `page-copywriting`, если у агента нет контекста и смысловой карты. Для одной страницы всё равно нужны аудитория, оффер, proof и CTA.

## Маршрут A: есть контекст и смыслы

1. `seo-content-brief`, если страница должна получать органический трафик.
2. `page-copywriting`, чтобы написать черновик.
3. `conversion-copy-review`, чтобы усилить ясность, доказательства и CTA.
4. `russian-copy-polish`, чтобы сделать финальный русский текст.

## Маршрут B: контекста нет

1. `product-marketing-context`
   - быстро собрать продукт, аудиторию, JTBD, alternatives, proof, CTA;
   - если данных мало, создать `Assumptions`.
2. `meaning-positioning`
   - сформулировать оффер, message hierarchy, objections, hero-смыслы.
3. `seo-content-brief`, если страница SEO-зависимая.
4. `page-copywriting`.
5. `conversion-copy-review`.
6. `russian-copy-polish`.

## Page brief минимум

- тип страницы;
- целевая аудитория;
- цель страницы;
- основной CTA;
- поисковый интент, если есть;
- ключевой оффер;
- доказательства;
- objections;
- ограничения по тону и длине.

Если нет хотя бы аудитории, оффера, CTA и proof/рисков proof, агент должен сначала создать краткий context + meaning, а не писать финальный текст.

## Output

- page brief recap;
- meta title, если SEO-page;
- meta description, если SEO-page;
- H1;
- hero variants;
- рекомендуемый hero;
- секции страницы;
- CTA;
- FAQ;
- блоки доверия;
- рекомендации по внутренним ссылкам;
- notes по недостающим proof.
