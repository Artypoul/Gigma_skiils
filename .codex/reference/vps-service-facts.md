# Карта фактов сервиса «Твой ВПС»

Опора для роли **консультант по подписке сервиса «Твой ВПС»**: что это, как человек управляет подпиской и подключается, и **как передать проблему Артёму**. Не выдумывать — если факта тут нет, сверяться с кодом кабинета (`vps_artpol`) или эскалировать.

## Формулировки и правовая рамка (читать первым)

В РФ действуют ограничения на рекламу и популяризацию средств обхода блокировок. Поэтому позиционирование и слова консультанта — строго нейтральные:

- Продукт — **сервис личного доступа по подписке** («Твой ВПС» = личный приватный **сервер**). **Не** называть его «VPN для обхода блокировок», **не** описывать как средство обхода блокировок или доступа к запрещённым ресурсам.
- Консультант помогает с двумя вещами: **управление подпиской** (статус, оплата, продление, аккаунт) и **техническая установка/настройка приложения** для подключения к своему доступу.
- **Не обсуждать:** обход блокировок, доступ к заблокированным/запрещённым ресурсам, «как смотреть запрещённое», «сменить страну, чтобы открыть X». На такие просьбы — вежливо не вовлекаться и оставаться в рамках «подписка + установка приложения».
- Нейтральный тон: «приложение», «подключение к сервису / к своему доступу», «подписка», «личный сервер». **Избегать** слов «обойти», «разблокировать», «обход», позиционирования «VPN».
- ⚠ Это не юридическая консультация. Финальные формулировки утверждает владелец сервиса.

## Что это

«Твой ВПС» — **сервис личного доступа по подписке** (личный приватный сервер). Технически работает на панели управления **PasarGuard**. У пользователя есть **личный кабинет**; вход — **по номеру телефона, без пароля («вход звонком»)**: ввести номер → нажать «Войти звонком» → позвонить на показанный номер со своего телефона → вход подтвердится сам (кода нет). Подробно — в скиле vps-connect (Шаг 0). Чтобы подключиться, нужна **ссылка-подписка** (`subscription_url`): её копируют из кабинета (`/home`) и добавляют в приложение-клиент (HAPP и аналоги). Одна ссылка = все доступные серверы доступа.

## Тарифы и состояния подписки

- **Тарифы:** `free` (бесплатный) и `pro` (платный). Тариф виден в поле `note` у пользователя PasarGuard.
- **Статусы подписки** (PasarGuard): `active`, `disabled`, `limited`, `expired`, `on_hold`.

| Что видит пользователь | Статус | Что значит |
|---|---|---|
| Новый бесплатный | `active` + `note=free` | работает на бесплатном тарифе |
| Ждёт оплаты | `disabled` («Ожидая оплату») | заказ создан, оплата не прошла |
| Оплачен | `active` + `note=pro` | платный тариф активен |
| Закончился | `expired` | срок вышел — нужно продлить |
| Лимит исчерпан | `limited` | упёрся в лимит трафика тарифа |

Канон «бесплатный»: `нет PG-юзера` ИЛИ `status != active` ИЛИ `note == free`.

## Контекст текущего пользователя (что агент видит, чтобы отвечать предметно)

Отвечая авторизованному пользователю, агент читает **его же** данные (read-only, в его сессии) и персонализирует ответ:

| Что видит | Поле / источник |
|---|---|
| Имя | `getCounterparty(token)` → профиль контрагента |
| Статус подписки | `status` (`active`/`disabled`/`limited`/`expired`/`on_hold`) |
| Тариф | `note` (`free` / `pro`) |
| Срок действия | `expire` |
| Трафик | `used_traffic` / `data_limit` |
| Телефон | есть (он же логин в PG), но по умолчанию **маскированный** (`maskPhone`) — для ответа не нужен |

Источник статуса/тарифа/срока/трафика — `getResolvedPGUserForToken(token)`; имени — `getCounterparty(token)`.

**Границы (важно):**
- Только данные **текущего** пользователя в **его** сессии. Не искать чужих клиентов по телефону/имени — это другая (операторская) роль.
- Только **чтение**. Тариф не менять напрямую: продление/смена = **ссылка на оплату** (`createCheckoutLink`), не admin-выдача.
- Перс. данные (телефон) не светить в открытых каналах; по умолчанию — маска.

## Где что в кабинете (роуты `vps_artpol`)

| Раздел | Зачем пользователю |
|---|---|
| `/home` (главная) | статус подписки + **ссылка-подписка** для подключения |
| `/select-plan`, `/plan` | выбор и просмотр тарифа |
| `/buy`, `/payments` | оплата и история платежей |
| `/usage` | статистика трафика |
| `/support` | форма обращения (тема «Поддержка» / «Возврат») → письмо в сервис |
| `/profile` | профиль, выход |

> Точный публичный адрес кабинета подставляется на проде из окружения — используйте ту ссылку на кабинет, которую выдаёт сервис.

## Технические нюансы (для агента — когда эскалировать)

Это **сторона сервиса**, пользователь руками не чинит. Могут меняться — при сомнении эскалировать, не утверждать как вечную истину:

- Сейчас сервис работает через сервер в Германии (DE). Один регион.
- Отдельные конфиги из подписки на некоторых сетях могут не подниматься — если **конкретный** конфиг не подключается, пробовать **другой конфиг из той же подписки**.
- На части сетей скорость может проседать. Если стабильно медленно на всех конфигах — это вопрос к сервису → эскалация.
