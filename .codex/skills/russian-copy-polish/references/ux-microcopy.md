# UX microcopy

Microcopy explains what happens next and reduces anxiety in forms, buttons, errors, and empty states.

## Button rules

- Start with a verb when possible.
- Be specific about the action.
- Avoid vague «Отправить» unless the context is obvious.
- Use the user's goal, not the system action.

| Context | Weak | Better direction |
|---|---|---|
| Demo | Узнать больше | Посмотреть демо / Получить демо |
| Consultation | Отправить | Обсудить проект |
| Download | Скачать | Скачать шаблон |
| Trial | Начать | Создать аккаунт / Начать тест |
| Pricing | Подробнее | Рассчитать стоимость |

## Form helper text

Tell users:

- why you ask;
- what format is expected;
- when they will get a response;
- what will happen after submission.

Examples of directions:

- «Ответим в течение рабочего дня».
- «Ссылка придёт на почту».
- «Можно описать задачу в 2–3 предложениях».
- «Не публикуем и не передаём контакты».

## Error messages

Good error message:

1. says what happened;
2. says how to fix it;
3. keeps a calm tone.

Pattern:

```text
Не получилось отправить форму. Проверьте интернет-соединение или напишите нам на [contact].
```

## Empty states

Explain:

- what is empty;
- why;
- what the user can do now.

Pattern:

```text
Пока нет добавленных проектов. Создайте первый проект, чтобы собрать структуру сайта и тексты в одном месте.
```

## Risk notes

Do not invent response times, privacy promises, guarantees, or legal statements. Mark missing details as `[уточнить]`.
