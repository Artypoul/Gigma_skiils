# CTA checklist

CTA is part of the strategy, not just button text.

## CTA quality checks

- [ ] CTA matches the user's intent and readiness.
- [ ] Button text says what will happen next.
- [ ] Primary CTA is consistent across key sections.
- [ ] Secondary CTA exists when commitment is high.
- [ ] CTA appears after enough explanation and proof.
- [ ] Form fields match the value of the offer.
- [ ] CTA is not hidden behind vague language.

## CTA examples by intent

| Intent | Weak CTA | Better direction |
|---|---|---|
| Demo | Узнать больше | Получить демо / Посмотреть продукт |
| Consultation | Отправить | Обсудить проект / Записаться на консультацию |
| Trial | Начать | Начать бесплатный тест / Создать аккаунт |
| Download | Скачать | Скачать шаблон / Получить чек-лист |
| Pricing | Подробнее | Рассчитать стоимость / Посмотреть тарифы |
| Contact | Связаться | Написать нам / Получить ответ по проекту |

## Placement

- Hero: primary next step.
- After mechanism: next step for convinced users.
- After proof: high-intent CTA.
- FAQ/final: CTA with risk reduction.

## Friction audit

- Are there too many required fields?
- Does the user know response time?
- Is the next step clear?
- Are there trust signals near the CTA?
- Is there a lower-friction alternative?
