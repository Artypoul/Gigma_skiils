# Chain gates

Use this file to decide whether the next skill can start.

| Gate | Must exist before moving forward | If missing |
|---|---|---|
| Context → Meaning | segment, JTBD/task, alternatives, value proposition, CTA, proof/proof gaps | create a compact product context with assumptions |
| Meaning → IA | core message, differentiation, objections, proof map, hero direction | create a provisional meaning map |
| IA → Content/SEO | page inventory, sitemap, URL map, navigation, internal links | create a minimal sitemap and mark URL assumptions |
| SEO → Copy | intent, H1/H2, metadata, must-answer questions, CTA, proof requirements | create intent-first brief and mark `Needs validation` |
| Copy → Review | full draft, CTA, proof blocks, FAQ or reason why FAQ is not needed | write the missing block or mark it as out of scope |
| Review → Polish | P1/P2/P3 fixes or accepted draft | revise before polish if P1/P2 remain |
| Polish → Handoff | final copy, risk notes, launch checklist | do not mark ready |

Never block on secondary details if a useful draft can be produced with explicit assumptions.
